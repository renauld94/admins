- Full resume (2 pages)
- Executive summary (1 page)
- Notes and customization details

## Usage

1. Each resume version is written in Markdown first
2. Convert to PDF using pandoc or export to DOCX
3. Always include both formats in application:
   - PDF for viewing (main submission)
   - DOCX for ATS parsing

## File Naming Convention

```
[YourName]_Resume_[JobTitle]_[CompanyRef].pdf
[YourName]_Executive_Summary_[JobTitle]_[CompanyRef].pdf
```

Example: `Simon_Renauld_Resume_Director_AI_Analytics_19096.pdf`

## Tips for Creating New Versions

1. **Start with full resume**: Copy base version
2. **Analyze job description**: Extract keywords, requirements, nice-to-haves
3. **Reframe experience**: Use job's language while staying truthful
4. **Quantify everything**: Numbers speak louder than adjectives
5. **Human voice**: Write like you're talking to a person, not a robot
6. **Proofread**: Get a friend to read itâ€”they'll catch what you miss

## Quality Checklist

Before submitting any resume:

- [ ] No typos or grammatical errors
- [ ] All metrics double-checked and accurate
- [ ] Job keywords appear naturally throughout
- [ ] Sentences vary in length and structure (avoid AI patterns)
- [ ] Contact info correct for this application
- [ ] File name follows convention
- [ ] Both PDF and DOCX versions generated
- [ ] Tested with ATS checker (resume.io, jobscan)
- [ ] Someone else has reviewed it
- [ ] Saved backup copy with date

## ATS Optimization

Every resume should be:
- Clean formatting (no tables, columns, or fancy graphics)
- Standard fonts (Arial, Calibri, or similar)
- Clear section headers
- Bullet points for achievements
- Keywords from job description
- Consistent date formatting
